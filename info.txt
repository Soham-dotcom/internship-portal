1. PROJECT OVERVIEW
================================================================================

PROJECT NAME: SPIT Internship Management & Analytics Portal
PURPOSE: Centralized portal for managing student internship records, tracking 
         progress, analyzing company-wise hiring, generating groups, and 
         providing comprehensive analytics.

TARGET USERS: SPIT coordinators, mentors, faculty members

KEY CAPABILITIES:
- Bulk Excel import/export
- Individual student record management
- Smart group generation with validation
- Advanced analytics with multi-filter support
- Company-wise hiring analysis

- Real-time data visualization

4. INSTALLATION & SETUP
================================================================================

PREREQUISITES:
--------------
- Node.js (v14 or higher)
- npm (comes with Node.js)
- MongoDB (local or cloud instance)
- Git (optional, for version control)

STEP 1: CLONE/NAVIGATE TO PROJECT
----------------------------------
cd Internship_portal

STEP 2: INSTALL ROOT DEPENDENCIES
---------------------------------
npm install

This installs:
- concurrently (run backend + frontend together)
- nodemon (auto-restart backend)
- dotenv (environment variables)

STEP 3: INSTALL BACKEND DEPENDENCIES
------------------------------------
cd backend
npm install

This installs:
- express (web framework)
- mongoose (MongoDB ODM)
- xlsx (Excel processing)
- cors (cross-origin requests)
- dotenv (environment variables)

STEP 4: INSTALL FRONTEND DEPENDENCIES
--------------------------------------
cd ../frontend
npm install

This installs:
- react, react-dom (UI framework)
- react-router-dom (navigation)
- axios (HTTP client)
- recharts (charts)
- tailwindcss (styling)
- xlsx (client-side Excel parsing)

STEP 5: SETUP ENVIRONMENT VARIABLES
------------------------------------
Create .env file in root directory:

MONGODB_URI=mongodb://localhost:27017/spit-internships
PORT=5000
REACT_APP_API_BASE=http://localhost:5000/api

STEP 6: START MONGODB
---------------------
If using local MongoDB:
- Windows: Start MongoDB service
- Mac/Linux: mongod --dbpath /path/to/data

Or use MongoDB Atlas (cloud) and update MONGODB_URI

STEP 7: RUN THE APPLICATION
----------------------------
From root directory:
npm run dev

This starts:
- Backend on http://localhost:5000
- Frontend on http://localhost:3000

STEP 8: SEED SAMPLE DATA (OPTIONAL)
-----------------------------------
In a new terminal:
npm run seed-salary

This loads 30 sample students with salary data for testing.

5. DATABASE SCHEMA
================================================================================

COLLECTION: internships

SCHEMA STRUCTURE:
----------------
{
  // Student Information
  email: String (optional, default: '')
  name: String (optional, default: '')
  uid: String (REQUIRED, UNIQUE) - Primary Key
  branch: String (optional, enum: ['COMPS', 'EXTC', 'CSE', 'MCA', 'AIML', 'IT', 'MECH', 'ETRX', ''])
  
  // Internship Details
  internshipType: String (optional, enum: ['Off-Campus', 'On-Campus', 'College-Arranged', 'Self-Arranged', ''])
  companyName: String (optional, default: '')
  externalMentorName: String (optional, default: '')
  
  // Dates
  startDate: Date (optional, default: null)
  endDate: Date (optional, default: null)
  
  // Additional Fields
  documentLink: String (optional, default: '')
  status: String (optional, enum: ['pending', 'approved', 'in-progress', 'completed', 'rejected', ''], default: 'pending')
  
  // Analytics Fields
  salary: Number (optional, default: 0) - For salary analysis
  duration: Number (optional, default: 0) - Duration in months
  
  // Legacy Fields (for backward compatibility)
  stipend: String (optional, default: '')
  companyLocation: String (optional, default: '')
  internshipTitle: String (optional, default: '')
  remarks: String (optional, default: '')
  
  // Group Assignment Tracking
  assignedGroup: String (optional, default: null) - Group ID if assigned
  assignedGroupName: String (optional, default: null) - Group name for display
  
  // Timestamps
  submittedAt: Date (default: Date.now)
  createdAt: Date (auto-generated)
  updatedAt: Date (auto-generated)
}

KEY CONSTRAINTS:
----------------
- uid: UNIQUE (Primary Key)
- uid: REQUIRED (cannot be empty)
- All other fields: OPTIONAL (can be empty)

INDEXES:
--------
- Unique index on uid field
- Automatic timestamps (createdAt, updatedAt)

6. API ENDPOINTS (COMPLETE LIST)
================================================================================

BASE URL: http://localhost:5000/api

INTERNSHIPS ENDPOINTS:
----------------------
GET    /api/internships
       - Get all internships
       - Query params: branch, company, status, mentor, type, startDate, endDate, uid, name
       - All filters are CASE-INSENSITIVE
       - Returns: { success: true, data: [...], count: number }

GET    /api/internships/:id
       - Get single internship by MongoDB _id
       - Returns: { success: true, data: {...} }

POST   /api/internships
       - Create new internship record
       - Body: { email, name, uid, branch, ... }
       - Returns: { success: true, data: {...} }

PUT    /api/internships/:id
       - Update internship by MongoDB _id
       - Body: { ...fields to update }
       - Returns: { success: true, data: {...} }

DELETE /api/internships/:id
       - Delete internship by MongoDB _id
       - Returns: { success: true, message: '...' }

UPLOAD ENDPOINTS:
-----------------
POST   /api/upload/import
       - Import parsed Excel data to database
       - Body: { internships: [...] }
       - Uses UPSERT logic (update if UID exists, insert if new)
       - Returns: { success: true, inserted: number, updated: number, failed: number }

GET    /api/upload/template
       - Download Excel template file
       - Returns: Excel file download

ANALYTICS ENDPOINTS:
--------------------
GET    /api/analytics/summary
       - Get dashboard summary
       - Query params: branch, status (case-insensitive)
       - Returns: { success: true, data: {...} }

GET    /api/analytics/companies
       - Get company-wise statistics
       - Query params: branch, status (case-insensitive)
       - Returns: { success: true, data: [...] }

GET    /api/analytics/branches
       - Get branch distribution
       - Returns: { success: true, data: [...] }

GET    /api/analytics/status
       - Get status distribution
       - Returns: { success: true, data: [...] }

ADVANCED ANALYTICS ENDPOINTS:
------------------------------
GET    /api/advanced-analytics
       - Get comprehensive analytics with multi-filter support
       - Query params: minSalary, maxSalary, internshipType, branch, 
                       minDuration, maxDuration, company, minStudents, status
       - All filters are CASE-INSENSITIVE
       - Returns: { success: true, data: { overview, salaryDistribution, 
                   topCompaniesByHiring, topCompaniesBySalary, 
                   topCompaniesByImpact, branchStats, ... } }

GET    /api/advanced-analytics/filters
       - Get available filter options
       - Returns: { success: true, filters: { companies, branches, 
                   internshipTypes, statuses, salaryRange, durationRange } }

GROUPS ENDPOINTS:
-----------------
POST   /api/groups/generate
       - Generate student groups with smart validation
       - Body: { filters: {...}, groupSize: number, numGroups: number, 
                 randomize: boolean, assignToGroups: boolean }
       - Validates inputs (prevents impossible configurations)
       - Auto-balances groups for even distribution
       - Only includes unassigned students
       - Returns: { success: true, data: { groups: [...], totalStudents, 
                   totalGroups, assigned: boolean } }

POST   /api/groups/check-assignment
       - Check if students are already assigned to groups
       - Body: { uids: [...] }
       - Returns: { success: true, alreadyAssigned: [...], count: number }

POST   /api/groups/unassign
       - Unassign students from groups
       - Body: { uids: [...] }
       - Returns: { success: true, message: '...', count: number }

GET    /api/groups/list
       - List all existing groups
       - Returns: { success: true, data: [...], count: number }

POST   /api/groups/export
       - Export all groups to Excel
       - Body: { groups: [...] }
       - Returns: Excel file download

POST   /api/groups/export-single
       - Export single group to Excel
       - Body: { group: {...} }
       - Returns: Excel file download

POST   /api/groups/random-pick
       - Pick random unassigned students
       - Body: { filters: {...}, count: number }
       - Only picks students NOT in any group
       - Returns: { success: true, data: [...], totalAvailable: number, 
                   picked: number }

POST   /api/groups/export-random
       - Export randomly picked students to Excel
       - Body: { students: [...] }
       - Returns: Excel file download

MENTOR EDIT ENDPOINTS:
----------------------
GET    /api/mentor-edit/:uid
       - Get student record by UID
       - Returns: { success: true, data: {...} }

PUT    /api/mentor-edit/:uid
       - Update entire student record by UID
       - Body: { ...all fields }
       - Returns: { success: true, message: '...', data: {...} }

PATCH  /api/mentor-edit/:uid
       - Update specific fields by UID
       - Body: { ...fields to update }
       - Returns: { success: true, message: '...', data: {...} }

POST   /api/mentor-edit
       - Create new student record
       - Body: { ...all fields }
       - Returns: { success: true, message: '...', data: {...} }

DELETE /api/mentor-edit/:uid
       - Delete student record by UID
       - Returns: { success: true, message: '...', data: {...} }

HEALTH CHECK:
-------------
GET    /api/health
       - Server health check
       - Returns: { status: 'OK', message: 'Server is running' }

11. TROUBLESHOOTING
================================================================================

ISSUE: "Cannot find module 'express'"
SOLUTION: Run "npm install" in backend directory

ISSUE: "Port 3000 is already in use"
SOLUTION: 
- Find process: netstat -ano | findstr :3000
- Kill process: taskkill /PID <pid> /F
- Or change port in frontend/package.json

ISSUE: "MongoDB connection error"
SOLUTION:
- Check if MongoDB is running
- Verify MONGODB_URI in .env file
- Check MongoDB service status

ISSUE: "No unassigned students found"
SOLUTION:
- Students are already in groups
- Go to Group Generator → View Existing Groups
- Click "Unassign All" on groups
- Try again

ISSUE: Excel upload shows "failed" records
SOLUTION:
- Check error messages in response
- Common issues:
  * Missing UID
  * Invalid date format
  * Invalid branch name (must match enum)
- Fix Excel and re-upload

ISSUE: Filters not working
SOLUTION:
- All filters are now case-insensitive
- Try different letter cases
- Check browser console for errors
- Verify backend is running

ISSUE: Charts not displaying
SOLUTION:
- Check if Recharts is installed: npm install recharts
- Verify data is loaded (check Network tab)
- Check browser console for errors

ISSUE: Group generation error
SOLUTION:
- Check if input is valid (enough students)
- Read error message for suggestions
- Try different group size/number combination

15. KEY CONCEPTS & ARCHITECTURE
================================================================================

DATA FLOW:
----------
1. User uploads Excel → Frontend parses → Sends to Backend
2. Backend validates → UPSERTs to MongoDB → Returns results
3. Frontend displays results → User can view/edit

UPSERT LOGIC:
-------------
- UID is the PRIMARY KEY (unique identifier)
- If UID exists in database: UPDATE entire record
- If UID is new: INSERT new record
- No duplicates possible (enforced by unique index)

GROUP ASSIGNMENT:
----------------
- Each student can belong to ONE group only
- assignedGroup field tracks group ID
- assignedGroupName field tracks group name
- Unassigned students have assignedGroup = null
- Group generation only includes unassigned students

CASE-INSENSITIVE FILTERS:
-------------------------
- All filters use RegExp with 'i' flag
- Exact matches: new RegExp(`^${value}$`, 'i')
- Partial matches: new RegExp(value, 'i')
- Works for: Branch, Status, Type, Company, etc.

SMART GROUP VALIDATION:
-----------------------
- Validates: totalStudents >= (numGroups × groupSize)
- Provides helpful error messages
- Suggests valid alternatives
- Auto-balances for even distribution

CHART LIBRARY:
--------------
- Recharts for all visualizations
- Responsive containers
- Interactive tooltips
- Professional legends
- Color-coded data
